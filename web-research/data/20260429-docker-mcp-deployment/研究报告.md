## 一、三种 Docker 部署方式

### 方式 1：手动 `docker run`（最通用）

所有官方和主流 MCP Server 都支持 stdio 模式的 Docker 运行：

```bash
docker run -i --rm <镜像名>
```

**核心参数说明**：
- `-i`：保持 stdin 打开，MCP 通过 stdio 通信
- `--rm`：容器退出后自动删除，避免孤儿容器
- `--init`（推荐）：插入 Tini 处理僵尸进程和信号转发

### 方式 2：Docker Desktop MCP Toolkit（最简单）

Docker Desktop 4.x+ 内置 MCP Toolkit，一键安装和管理：

1. Docker Desktop → Settings → 勾选 "Enable MCP Toolkit"
2. 在 MCP Catalog（https://hub.docker.com/catalogs/mcp）浏览 100+ 验证过的 MCP Server
3. 点击 Install + Connect，自动配置 Claude Desktop / Cursor / VS Code

客户端配置只需一行：
```json
{
  "mcpServers": {
    "MCP_DOCKER": {
      "command": "docker",
      "args": ["mcp", "gateway", "run"]
    }
  }
}
```

### 方式 3：Docker Compose（多服务编排）

适合同时运行多个 MCP Server 或需要数据库的服务（如 MetaMCP）。

## 三、具体部署配置

### 3.1 GitHub MCP Server

```json
{
  "mcpServers": {
    "github": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-e", "GITHUB_PERSONAL_ACCESS_TOKEN",
        "ghcr.io/github/github-mcp-server"
      ],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxx"
      }
    }
  }
}
```

可选控制工具集：添加 `-e GITHUB_TOOLSETS=repos,issues,prs` 只启用特定功能。

### 3.2 Playwright MCP Server

```json
{
  "mcpServers": {
    "playwright": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm", "--init", "--pull=always",
        "mcr.microsoft.com/playwright/mcp"
      ]
    }
  }
}
```

长驻 HTTP 模式（适合多客户端）：
```bash
docker run -d --rm -p 8931:8931 \
  mcr.microsoft.com/playwright/mcp \
  --port 8931 --host 0.0.0.0
```

### 3.3 官方 Filesystem Server

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-v", "/你的项目目录:/projects",
        "mcp/filesystem",
        "/projects"
      ]
    }
  }
}
```

> ⚠️ 官方 Docker 镜像要求绑定挂载路径在 `/projects` 下。

### 3.4 官方 Time Server（带时区）

```json
{
  "mcpServers": {
    "time": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-e", "LOCAL_TIMEZONE=Asia/Shanghai",
        "mcp/time"
      ]
    }
  }
}
```

### 3.5 MetaMCP（Web 管理平台）

需要 PostgreSQL，使用 Docker Compose：

```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: metamcp
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: changeme
    volumes:
      - postgres_data:/var/lib/postgresql/data

  metamcp:
    image: ghcr.io/metatool-ai/metamcp:latest
    ports:
      - "12008:12008"
    depends_on:
      - postgres
    environment:
      DATABASE_URL: postgresql://postgres:changeme@postgres:5432/metamcp
      APP_URL: http://localhost:12008
      BETTER_AUTH_SECRET: changeme
```

### 3.6 Docker 管理 MCP（AI 操作 Docker）

```json
{
  "mcpServers": {
    "docker-mgmt": {
      "command": "docker",
      "args": [
        "run", "-i", "--rm",
        "-v", "/var/run/docker.sock:/var/run/docker.sock",
        "mcp-server-docker"
      ]
    }
  }
}
```

> ⚠️ 挂载 Docker socket 等于授予 AI 完整的 Docker 控制权（包括 root 权限），仅在受信环境使用。

## 四、传输方式对比

| 传输方式 | 原理 | 适用场景 | Docker 参数 |
|----------|------|---------|------------|
| **stdio** | 通过 stdin/stdout 管道通信 | 单客户端（Claude Desktop/Cursor） | `-i --rm` |
| **SSE** | HTTP Server-Sent Events | Web 应用、多用户 | `-p 端口:端口` |
| **HTTP (Streamable)** | 标准 HTTP 请求/响应 | 微服务、K8s | `-p 端口:端口` |

**选型建议**：
- 个人开发 → **stdio**（最简单，`-i --rm` 即可）
- 团队共享 → **HTTP**（长驻服务，多人访问）
- 生产环境 → **K8s + HTTP**（microsoft/mcp-gateway）

## 五、安全注意事项

1. **永远不要硬编码 Token** — 用环境变量传递 API Key 和 PAT
2. **Docker socket 挂载 = root 权限** — 仅在受信环境使用 mcp-server-docker
3. **路径挂载限制** — Filesystem Server 限制在 `/projects` 下，防止越权访问
4. **官方镜像安全** — GitHub/Microsoft 镜像使用 distroless/slim 基础镜像，非 root 运行
5. **定期更新镜像** — 使用 `--pull=always` 确保最新版本
