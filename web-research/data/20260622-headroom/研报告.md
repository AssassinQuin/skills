## 二、解决的痛点

AI agent 的 context window 被工具输出/日志/RAG/文件/对话历史挤爆。**Headroom 主张压缩在 LLM 之前**——same answers, fraction of the tokens。

## 三、4 种部署形态（最完整产品矩阵）

| 形态 | 命令 |
|---|---|
| Library | `from headroom import compress` |
| Proxy | `headroom proxy --port 8787` |
| Agent wrap | `headroom wrap claude\|codex\|cursor\|aider\|copilot` |
| MCP server | `headroom mcp install`（compress/retrieve/stats）|

## 四、6 种压缩算法

```
CacheAligner → ContentRouter → CCR
               ├─ SmartCrusher   (JSON)
               ├─ CodeCompressor (AST: Py/JS/Go/Rust/Java/C++)
               ├─ Kompress-base  (HF 文本模型)
               └─ Image compression (ML router, 40-90% reduction)
```

CCR（Compression with Cached Retrieval）= 可逆压缩，originals 本地缓存，LLM 通过 `headroom_retrieve` 按需取回。

## 六、独特功能

1. **`headroom learn`**——挖 failed session 写 corrections 到 CLAUDE.md/AGENTS.md/GEMINI.md
2. **Output token reduction**——不只压 input，也压 output（"be terse" steering + effort routing）
3. **跨 agent 共享 memory**——Claude/Codex/Gemini 共用 store，auto-dedup
4. **Verbosity 自动学习**——`headroom learn --verbosity` 分析用户中断行为自动选级

## 七、与 RTK 的关系（重要）

**Headroom 打包 RTK 二进制作为 shell 输出重写的一部分**。README 明确致谢 RTK 团队。可选 `HEADROOM_CONTEXT_TOOL=lean-ctx` 切换。

## 十、缺点（按严重度）

### 🔴 高严重度

1. **Prefix cache 破坏**（最严重）：
   - #1042 Canonical re-serialization → 17/18 prefix cache bust（`cache_read=0`）
   - #1011 Headroom 让 Reasonix + DeepSeek v4 出现 2-3x 更多 cache misses
   - 讽刺：CacheAligner 卖点就是稳定前缀，但自身实现破坏了它
2. **冷启动灾难**：
   - #1146 274MB ONNX 在 request path 内联下载 → 30s timeout → silent fail-open
3. **Windows 二等公民**：
   - #1202 `headroom learn` 在 Windows 完全坏（缺 encoding="utf-8"）
   - #1069 X-Headroom-Project 拒绝中日韩文目录名
4. **关键路径 crash**：
   - #1145 UnboundLocalError 'ccr_workspace_key' 在 `--no-ccr-inject-tool` 崩所有请求
   - #1092 wrap --memory 无 torch 时崩

### 🟡 中严重度

5. provider 集成回归：
   - #1107 wrap codex 忽略 OPENAI_BASE_URL
   - #1039 GitHub Copilot models 被 proxy 过滤
   - #1189 Codex 内置 image generation 在 proxy 失败
   - #1058 sticky_betas 让所有 Claude Code 请求 usage credits 错误
6. 硬编码假设：#1038 DeepSeek-V3 硬编码 128K（应 1M）
7. packaging bug：#1232 [code] extra 静默禁用代码压缩
8. 副作用外溢：#1235 RTK guidance 写到 project AGENTS.md
9. MCP retrieve bug：#1213/#1095 hash 长度不匹配

### 🟢 策略争议

10. 复杂度爆炸——6 算法 × 4 形态 × 6 agent × 5 provider，回归测试难覆盖
11. "全都要"风险——rUv 类的广度扩张
