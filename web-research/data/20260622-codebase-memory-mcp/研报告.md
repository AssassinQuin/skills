## 二、解决的痛点

AI agent 理解大型代码库的**成本与准确性**：从"Read 整文件 + grep + LLM 推理 + token 爆炸"变成"图谱查询 sub-ms 返回 + 99% token 省"。

## 三、技术架构

```
单静态二进制（C，~15MB）
├── Tree-sitter (158 语法)
├── Hybrid LSP (9 语言类型感知：Java/Kotlin/Rust/C/C++/Py/TS/Go/C#/PHP)
├── SQLite (WAL) + Cypher 引擎
├── mimalloc + simhash
├── 自研 HTTP server (v0.8.1)
├── Semantic Search (Nomic 向量)
└── Leiden 社区检测 (v0.8.0)
```

零运行时依赖（所有库 vendored）。分发渠道全覆盖：npm/PyPI/Homebrew/AUR/Chocolatey/Scoop/Winget/Go。

## 七、缺点（按严重度）

### 🔴 高严重度

1. **Windows 支持糟糕**（最大痛点）：
   - #529 `WaitForSingleObject` 在匿名管道 `WAIT_FAILED` → stdio 握手挂死
   - #422 `search_code` **所有 Windows 用户零结果**（硬编码 `/bin/bash` + GNU grep）
   - #423 CLI 参数解析对 Windows 路径错误
   - #437 OpenCode MCP 配置 shape 错
2. **大型 C++ 索引崩溃**：
   - #410 索引 freezes（22423 defs 的 .hpp）
   - #424 SEGV（tree-sitter `0xffffffffffffffff` sentinel 未检测）
   - #432 模板化 C++ SEGV（`targs[16]` uninitialized stack）
3. **缓存损坏循环**：
   - #439 单损坏 DB 导致整个 server SIGABRT 启动（24ms 崩）→ 影响所有 sessions → 崩溃中断 in-flight 写 → 产生更多损坏 → 自持续
   - 用户报告 39/88 个 DB 损坏
   - #67 的修复路径在 v0.7.0 仍复现

### 🟡 中严重度

4. 静默数据丢失：
   - #411 `mode=moderate` 静默丢子树（Python `tools/` 47 文件消失）
   - #511 `get_code_snippet` 非 UTF-8 返回无效 JSON → 客户端挂死
5. #446 `.github/` 硬编码排除 → GitHub Actions workflows 不可见
6. #406 stdio 父进程死后 server 变孤儿

### 🟢 低严重度

7. #547 Release 只保留最新版（v0.5.7-v0.8.0 全 404）下游 pin silent fail
8. #529 多语言 .gitignore 不级联
9. `.git/info/exclude` 不识别
