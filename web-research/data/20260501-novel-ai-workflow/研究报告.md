## 四、核心方法论

### 4.1 4T写作Prompt框架（最系统）

- **Think**: 深度思考与全局设计（世界观/角色/冲突）— 5轮渐进式对话
- **Tissue**: 精细化结构编织（每章详细结构+爽点+伏笔）
- **Typesetting**: 智能评分与资源优化（S/A/B/C级章节分类）
- **Text**: 高质量内容输出（10万字=25章×4000字为一个单元）

### 4.2 2BCDE冲突设计框架

- Collision（冲突）+ Drive（驱动力）+ End（终点）+ Barriers（障碍）+ Bridge Characters（桥梁角色）

### 4.3 分层生成技术

骨架层 → 血肉层 → 润色层 → 校正层

### 4.4 AI三大缺陷（工程级方法论）

- 结构过平、情绪过直、节奏过均
- "AI味"检测特征：句式平滑度异常、逻辑过渡异常自然、高频词集中、信息密度均匀化

### 4.5 长篇控制方案

- 角色一致性系统：人设数据库 + 行为约束 + 记忆锁定
- 章节分块 + 线索回收管理 + 主线稳定器
- 主流方案：RAG + 上下文快照 + 专门化Agent分工

## 五、平台适配经验

### 番茄小说

- AI版权归属作者，平台不查AI来源
- 字数节点：签约2万字，首秀10万字，全勤需10万字+听读≥200+日更4000-6000字
- 每章2050-2300字利于有声小说和冲奖励
- 首秀前建议存稿14万字

### 起点

- 要求更高，AI文较难达到起点水平
- 重视铺垫、强逻辑、人设深度

### 纵横

- 兼顾节奏与剧情张力
- 主打玄幻、仙侠品类

### 通用注意

- 所有平台需注意"降AI率"
- 避免被识别为低质批量生成

## 六、共性工作流模式

所有成熟方案的共性流程：

```
题材设计 → 大纲搭建 → 章节创作 → 审阅润色 → 去AI味 → 质量门禁 → 分发落盘
```

核心挑战均为「长篇记忆/遗忘」和「去AI味」。
