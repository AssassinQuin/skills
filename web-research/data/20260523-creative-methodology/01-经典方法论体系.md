# 经典创意方法论体系

---

## 1. TRIZ（发明问题解决理论）

**创始人**: Genrich Altshuller（苏联工程师），1946-1985
**核心发现**: 分析300万+专利后发现，创新模式可复现、非随机

### 核心机制
- **两种矛盾类型**:
  - 技术矛盾：改善参数A导致参数B恶化 → 用39个消除原理
  - 物理矛盾：系统有相反需求 → 用分离原理
- **40条发明原理**: 分割、取出、局部质量、非对称、合并、嵌套、反馈等
- **流程**: 识别问题 → 对比TRIZ通用问题 → 找解决方案 → 映射回具体问题

### 冷门高级工具
- **九屏法**: 时间(过去/现在/未来) × 空间(子系统/系统/超系统) 九维度分析
- **小人法**: 用"小人"模拟系统部件交互，直观发现矛盾
- **ARIZ算法**: 最复杂的问题解决算法，适合极端困难矛盾

### 适用场景
技术矛盾/系统创新/需要根本性突破时
**学习曲线**: 陡峭（需理解40原理+矛盾矩阵）

- https://zhuanlan.zhihu.com/p/613377697
- https://www.theiet.org.cn/Data/View/722

## 2. SCAMPER（七动词改造法）

**起源**: Alex Osborn 1953（BBDO广告公司联合创始人，头脑风暴发明者）→ Bob Eberle 1971整理为助记词
**本质**: 侧向思维技术，挑战现状、探索新可能

### 七个维度 + 操作问题
| 动词 | 问题 | 商业案例 |
|------|------|---------|
| **Substitute** 替代 | 什么能替换以改善？ | IKEA用快生木材替代传统木材 |
| **Combine** 组合 | 什么可合并创造协同？ | 手机=无线电+电话+相机 |
| **Adapt** 适应 | 什么可从其他领域借鉴？ | Ford从屠宰场借鉴装配线 |
| **Modify** 修改 | 什么可放大或缩小？ | 迷你化产品线 |
| **Put to another use** 他用 | 还能用于何处？ | 烘焙苏打的100种用法 |
| **Eliminate** 消除 | 什么可移除？ | Apple去掉DVD光驱 |
| **Rearrange/Reverse** 重排/逆转 | 什么可重排或反转？ | 塑料袋→回归纸袋 |

### 在设计思维中的定位
SCAMPER不在早期发现阶段，而在Double Diamond第三阶段（Develop）

### 高级变体
- 整合思考（同时应用多个SCAMPER技术）
- 迭代应用（连续反馈循环）
- 系统思维视角（不只看产品，考虑整个系统含人力因素）

### 来源
- https://ixdf.org/literature/topics/scamper
- https://www.designorate.com/scamper-technique-examples-and-applications/

## 3. 六顶思考帽（Six Thinking Hats）

**创始人**: Edward de Bono, 1985
**核心机制**: 平行思考——团队同时从一个视角审视问题，避免对抗式讨论

### 六顶帽子
| 颜色 | 角色 | 关注点 |
|------|------|--------|
| 白帽 | 事实/数据 | 我们知道什么？缺少什么信息？ |
| 红帽 | 情感/直觉 | 我的感觉是什么？直觉告诉什么？ |
| 黑帽 | 谨慎/风险 | 什么地方可能出错？ |
| 黄帽 | 乐观/益处 | 有什么价值？为什么能行？ |
| 绿帽 | 创意/替代 | 还有什么替代方案？ |
| 蓝帽 | 流程控制 | 我们在用什么思考流程？下一步？ |

### 方法组合建议
绿帽发散(SCAMPER生成) → 黑帽收敛(风险评估) → 黄帽确认(价值验证)

## 4. 侧向思维（Lateral Thinking）

**创始人**: Edward de Bono, 1967
**核心**: 打破既有模式，从侧面迂回解决问题

### 关键技术
- **随机输入(Random Entry)**: 从字典随机翻词，强制关联当前问题
- **PO挑衅**: 故意陈述荒谬命题作为起点（如"PO：客户应该付钱让我们拿走产品"），绕过大脑自我审查
- **挑战假设**: 列出所有假设并逐个质疑
- **概念扇**: 问题→概念→替代方案的扇形展开
- **类比与故事**: 用跨领域类比激发新视角

## 5. 方法选择决策树

```
问题类型？
├── 技术矛盾（改善A导致B恶化）→ TRIZ
├── 快速头脑风暴/改进现有产品 → SCAMPER
├── 模糊/用户中心/邪恶问题 → 设计思维
├── 需要全面评估已有方案 → 六顶思考帽
├── 突破瓶颈/范式转换 → 侧向思维/PO
├── 多维度系统性探索 → 形态分析法
└── 根本性创新/推翻假设 → 第一性原理
```

### 方法互补矩阵
- 问题定义 → 白帽 → 侧向思维(挑战假设)
- 想法生成 → SCAMPER → 绿帽 → 随机输入
- 克服矛盾 → TRIZ → PO挑衅
- 评估想法 → 黑帽+黄帽
- 系统精炼 → TRIZ+形态分析
