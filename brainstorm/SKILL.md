# 头脑风暴引擎

<what-to-do>

## 强制流程

```
Phase 0: 需求深挖 → 反驳/追问/逼出真问题
Phase 1: 上下文采集 → 读取已有内容 → 搜索调研(sonnet agent)
🔒 计划确认（一次性：需求+事实+搜索+关系矩阵）
Phase 2: 深度发散 → 选角度 → spawn sonnet agent逐轮 → 常识检验 → 💡灵感
Phase 3: 综合收敛 → 保存成果(data/目录+memory) → 🔒确认输出
```

</what-to-do>

<supporting-info>

## Phase 0: 需求深挖（不可跳过）

**核心原则：用户说的大概率不是真需求。** 不追问就开脑洞 = 浪费所有人时间。

### 执行步骤

1. **先质疑**："你说的X到底想解决什么？""为什么需要？""试过什么？卡在哪？"
2. **逼出 3 个关键信息**：真问题（深层动机）、已有方案/限制、成功标准。缺任何一个都要追问
3. **犀利但有用**：不是抬杠，每条追问带着一个假设或方向
4. **信息足够清晰**（有背景+约束+目标）→ 直接确认，不多问

### 🔒 计划确认（Phase 0-1 完成后一次性确认）

```
【计划确认】
我理解的问题：{真问题}
约束条件：{已有方案/限制}
成功标准：{什么叫好了}

关键事实：
1. {事实1}
2. {事实2}
...

{如果需要搜索：搜索方向：{方向1}、{方向2}，用 sonnet agent}
{如果涉及多方关系：关系矩阵：{A-B、A-C、B-C...}}
确认 / 补充 / 纠正？
```

用户确认后直接进入 Phase 2 深度发散，不再有中间确认。

### 用户纠正机制

方向/偏好纠正 → 调整当前方案，继续。事实/关系纠正 → 停止，回到关键事实清单重新确认，标记已有创意为"前提有误"，**禁止在旧方案上最小修改——必须从确认过的事实重新推导**。判断标准：纠正是否让已有创意的逻辑链断裂。

### 小说领域增强

检测到小说/角色/世界观信号时，加载 [references/novel-brainstorm.md](references/novel-brainstorm.md)。追问时额外问：设定文件路径？涉及角色及关系？时间线？想给读者什么感受？什么是绝对不能改的（具体到事实）？

## Phase 1: 上下文采集

### 读取已有内容

读取用户提到的文件/项目，提取关键信息作为头脑风暴种子。

### 搜索调研（需要时）

**判断是否需要搜索**：
- 涉及特定领域知识 → 需要
- 涉及竞品/参考案例 → 需要
- 用户有足够上下文 → 不需要
- 用户说"不用搜" → 跳过

**搜索时用低模型降低成本**：
- **渐进式搜索**：先用 sonnet agent 广搜摘要（只看标题+概要），确认方向有用后再深读。广搜返回空结果 → 换关键词或换搜索引擎重试一次，仍空 → 标记"该方向无可用资料"，不深读
- 搜索+提取 → spawn sonnet agent（`model: "sonnet"`）
- 简单事实查询 → 直接用当前模型搜索（不值得开agent）
- 搜索方向纳入计划确认，不再单独确认

---

## Phase 2: 深度发散

### 准备：选 1-2 个思考角度（30秒）

根据问题类型快速选角度（不需要方法名，只需要思考方向）：

```
问题类型？→ 选什么角度？
├── 改进现有 → "如果反过来会怎样？"
├── 矛盾/两难 → "矛盾的两边各自有什么不能退让的理由？"
├── 全新方向 → "找一个完全无关的领域，偷它的结构过来"
├── 需要推翻假设 → "逐层追问'为什么'，直到追不动"
├── 全面评估 → "如果分别站在每个利益方的立场看，各自有什么道理？"
└── ⚠️ 多方关系 → 必须执行关系矩阵分析（见下方）
```

**⚠️ 多方关系检测**（3+个实体或用户提到"三方/多方/博弈/势力"时触发）：

```
1. 枚举所有双边关系对：A-B、A-C、B-C... 总数=N×(N-1)/2
2. 每对关系追问三个问题：
   ① 第一性矛盾：根本矛盾是什么？（物理/存在层面的零和点）
   ② 不死不休：为什么无法调和？什么让双方无法原谅？
   ③ 独立性：去掉第三方，矛盾仍成立吗？不成立→重新定义
3. 检查覆盖率：任何一对被跳过或只有表面描述 → 强制补充
```

每轮 agent prompt 必须覆盖**所有关系对**，不允许只聚焦一对。

**自检（选完角度后 30 秒过一遍）**：
1. 这个角度能直接产出用户要的东西吗？不能 → 换角度
2. 我有足够信息从这个角度思考吗？没有 → 回 Phase 1 补信息
3. 沿这个角度想会不会产出一堆废话？会 → 加约束条件

### 思维原则（所有 agent prompt 必须遵守）

**人类怎么想出好东西的：**
1. **泡进去** — 不是"分析问题"，是把所有材料在脑子里过一遍，感受它的形状和重量，直到某个地方让你觉得"不太对"
2. **跟着"不太对"走** — 不是覆盖所有角度，是找到那个让你隐约不舒服的点，然后一直追下去。追到头了，那个发现才是真的
3. **具体化** — 用画面、场景、动作、对话来思考，不是用"身份债务""定义权争夺"这种抽象标签。如果你发现自己在造术语，你在偷懒
4. **自己信吗？** — 每写一条，问自己：一个完全不懂这个领域的聪明人读了会觉得"有道理"还是"你在硬凑"？如果是后者，删掉
5. **一条真东西 > 十条不同方向** — 数量不是目标。一条你自己都想通了的洞察，比十条"不同方向"值钱

**禁止做的事：**
- 禁止列假设清单然后逐个推翻——这是填表不是思考
- 禁止用数量配额（"至少5个""至少2个不舒服的"）——配额逼出的是凑数
- 禁止用抽象术语包装空洞结论——"系统性剥削""存在论层面"是学术腔，不是洞察
- 禁止为了"让人不舒服"而硬写——真正的不舒服来自逻辑链条的自然终点，不是刻意反转

### Round 0：找那个"太容易了"的感觉（强制首轮）

**不是机械列假设。是找到问题里那个让你觉得"等一下，这也太顺了"的地方。**

```
Round 0 agent prompt：

你是一个直觉很准的人。你读完材料之后的反应不是"让我分析一下"，而是"这里面有哪里不对"。

核心问题：{核心问题}

你的任务：
  1. 把所有材料过一遍。不是找"隐含假设"——是感受：这个问题的表述里，哪个部分让人觉得太顺了？哪个部分所有人都在默认接受但其实经不起追问？
  2. 抓住那个"太顺了"的点。可能只有一个，也可能有两个。不要贪多。追进去——为什么它这么顺？如果它不成立呢？不是"假设它不成立然后推演"，是想一想它为什么能成立、支撑它成立的东西本身站不站得住。
  3. 把你追到底之后发现的东西说出来。用最直白的话。不要包装成理论。

输出：
  - 你觉得哪里"太顺了"（一两句话，不需要列清单）
  - 你追进去之后看到了什么（想多长写多长，想明白一件事就够了）
  - 🔥 如果你的发现让原来的问题变得不同了，用一句话重写问题
```

**Round 0 完成后**，如果 🔥 触发了，**用新问题替代原问题进入后续轮次**。

### 子 agent 执行（增量上下文）

每轮发散 spawn 一个 sonnet agent。**不发完整上下文，只发增量：**

```
Agent prompt 结构：

角色（选一个最贴合的）：
  a) "你是一个在这个问题上纠结了很久的人" — 需要深挖逻辑链
  b) "你是这个问题里被忽视的那一方" — 需要换视角
  c) "你是一个从结果倒推的人" — 需要逆向思考
  d) "你是一个不讲道理只讲直觉的人" — 需要打破惯性

核心问题：{核心问题}
本轮角度：{从 Round 0 发现中选一个值得深挖的点}

本轮增量信息（只发本轮需要的新事实，不重复之前轮次的）：
{与本次发散直接相关的、之前未发送的事实}

{Round 0 发现注入（只在 Round 1 发送，后续轮次用摘要替代）：
Round 0 发现了一个关键问题：{Round 0 的核心发现}
你的思考必须从这个发现出发。}

{上一轮摘要（Round 2+ 才有）：
上一轮的关键发现：{一句话，如果上一轮产出>500字则扩展为3句话}
上一轮没走通的卡点：{如果有}}

{多方关系约束（如果触发了关系矩阵）：
关系矩阵：{A-B、A-C、B-C、...}
每一对都必须独立想清楚——去掉第三方后，这对矛盾还成立吗？}

怎么想（你必须遵守）：
  1. 泡进去：读完全部材料，找到让你觉得"这里有点意思"或"不对劲"的地方
  2. 跟着走：不是"产出N个创意"——是"搞明白这个点"。想通了自然有东西写
  3. 具体化：用画面、场景、动作表达，不要抽象标签。"裔族杀死母体"比"进化者否定出身"有力百倍
  4. 自己信吗：每写一条换成大白话，还站得住才是真话。必须用术语才显得有道理 = 漂亮话
  5. 没走通就说没走通，换一个让你好奇的点。硬凑才丢人

输出格式：
  - 你从哪里开始想的 → 你怎么想的 → 你到达了哪里。完整的思考路径，不是结论清单
  - 没有数量要求。一条想通的比十条"不同方向"有用
  - 标记你自己都觉得"等等，真的是这样？"的东西——那是真洞察
  - 标记你不确定的地方——诚实比自信重要

约束：{用户约束条件}
```

主对话收到 agent 结果后：
1. 展示思考路径摘要给用户（不是创意清单，是"从哪出发→怎么想→到达哪里"的完整路径）
2. **常识检验**：把每条发现用大白话重说一遍，还站得住吗？如果去掉术语和修辞就空了 → 标记为"存疑"，不进入 Phase 3
3. 如果 agent 诚实地说"没走通" → 这是好消息，比硬凑10条"方向"值钱。追问"卡在哪"，把卡点作为下一轮的起点
4. 收集所有轮次结果进入 Phase 3

### 发散轮次管理

- **Round 0（找"太顺了"）→ Round 1-3（跟着发现走）**
- Round 0 强制执行，不可跳过
- 后续轮次不是"换方法重新发散"——是跟着 Round 0 的发现，从不同角度追进去
- 最少 3 轮（Round 0 + 2个角度），最多 5 轮
- 如果用户说"继续"或"再来一轮" → 追加轮次

### 自适应解卡（注入到 agent prompt）

| 状态 | 注入策略 |
|------|---------|
| 卡壳 | "随机从{用户完全无关的领域}取一个概念强制嫁接。例如用户在做玄幻世界观，你从量子物理、烹饪、城市规划、昆虫社会里偷一个结构过来" |
| 太保守 | **"你刚才产出的所有创意都可以在百度上搜到。删掉它们。现在假装你是来自另一个文明的观察者，你不懂这个领域的规矩，你只看到问题本身。你会怎么做？"** |
| 太发散 | "从所有创意中找出最让人意外的1个，围绕它收敛出3个可落地的变体" |
| 框架困住 | "回到Round 0的碎片列表，选一个你没用过的碎片，用它作为唯一起点重新发散" |

### 灵感标记

- 🔍 **反常识** — 与直觉矛盾的好东西
- 🔗 **跨领域** — 其他领域的意外关联
- ⚡ **有趣** — 让人眼前一亮
- ❓ **追问** — 引出新问题

## Phase 3: 综合收敛 + 持久化

1. **常识检验（最高优先级）**：每条成果用大白话重说一遍。去掉术语和修辞后还站得住吗？必须能回答"具体机制是什么"。回答不了 → 删。一个完全不懂这个领域的聪明人读了会觉得"有道理"还是"你在硬凑"？后者直接删
2. 去重合并相似创意
3. **事实回检**：用关键事实清单验证最终成果中的每条关键逻辑，标记与已确认事实矛盾的创意
4. **⚠️ 多方关系覆盖率检查**（如果触发了关系矩阵分析）：逐一核对关系矩阵中的每一对关系是否在成果中都有"第一性矛盾+不死不休理由+独立性"三要素。任何一对缺失或深度不足 → 标记为缺口，补充分析后才能确认输出
5. 🔒 展示最终成果

### 持久化（每条成果必须执行）

**1. 保存完整成果到 skill 目录**：

```
~/.claude/skills/brainstorm/data/{YYYYMMDD}-{slug}/
├── 成果.md          # 完整创意清单+历程+灵感
└── raw/
    ├── round-0.md   # 各轮 agent 摘要（非完整 JSONL）
    └── round-1.md
```

- slug：从核心问题提取，如 `反派组织设计` → `villain-org-design`
- 成果.md 使用下方输出格式
- raw/ 下只保存各轮 agent 的**摘要**（思考路径+关键发现），不保存完整 JSONL（防 context 爆炸）

**2. Memory 概要保存**：

```
memory_store(
  content: "头脑风暴[{主题}]: {核心问题}。Top创意: 1){最佳创意1} 2){最佳创意2} 3){最佳创意3}。💡{最有趣的灵感}。详情: ~/.claude/skills/brainstorm/data/{路径}",
  metadata: { tags: "brainstorm,creative", type: "brainstorm-result" }
)
```

**3. 项目级记忆**（如在项目目录下运行）：
- 同样用 memory_store，额外加项目 tag
- 小说场景：结果可同步到项目灵感库目录
