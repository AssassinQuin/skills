# 创意捕捉引擎

## 核心理念

创意不是"生产"出来的，是"捕捉"到的。人的创意过程不是"收集信息→应用框架→产出创意"，
更像是：问题在脑子里 → 做别的事时突然想到一个奇怪的连接 → 追那个连接 → 发现它能解决问题 → 回头验证。

本引擎模拟这个过程。

## 状态机

```
DIGGING ──→ INCUBATING ──→ CAPTURING ──→ FORGING ──→ SHIPPING
  (深挖)       (碰撞孵化)      (捕捉)       (锻造)       (交付)
                 ↑                  │
                 └── 不足时回退 ─────┘  （最多3轮）
```

---

## DIGGING：深挖真问题

**核心原则：用户说的大概率不是真需求。**

### 输入校验（最先执行）

以下场景不触发本 skill：技术实现/代码debug/文件编辑/数学计算/格式转换/数据库操作/系统配置/事实性问答/单一正确答案。
模糊边界：创意需求与技术信号并存 → 以创意需求为主触发，技术信号作为约束条件。

### 步骤

1. **先质疑**："你说的X到底想解决什么？""为什么需要？""试过什么？卡在哪？"
2. **逼出 3 个关键信息**：真问题（深层动机）、已有方案/限制、成功标准
3. **犀利但有用**：不是抬杠，每条追问带着一个假设或方向
4. 信息足够清晰 → 直接确认，不多问

异常：用户说"别问了" → 尊重，标注假设供纠正；回答矛盾 → 指出矛盾让用户选；追问2轮仍模糊 → 用已有信息+假设填充。

### 确认

```
【确认】
核心矛盾：{一句话描述问题的核心张力}
约束条件：{已有方案/限制}
成功标准：{什么叫好了}
关键事实：1. {} 2. {} ...
{假设（未确认）：{}}
确认 / 补充 / 纠正？
```

用户确认后进入 INCUBATING，不再有中间确认直到 SHIPPING。

### 小说领域增强

检测到小说/角色/世界观信号时，加载 [references/novel-brainstorm.md](references/novel-brainstorm.md)。追问时额外问：设定文件路径？涉及角色及关系？时间线？想给读者什么感受？什么是绝对不能改的？

### 用户纠正机制

- 方向/偏好纠正 → 调整当前方案，继续
- 事实纠正 → 停止。回到确认清单重新确认，已有创意标记"前提有误"，禁止最小修改，从确认过的事实重新推导
- 过程中用户反悔 → 中断当前状态，回 DIGGING 重新确认，已有产出标记"基于旧前提"

## INCUBATING：碰撞孵化

**核心操作：拿核心矛盾和一个随机不相关的事物碰撞。**

不看任何已有材料、不搜索、不查案例。纯粹让问题和完全不相关的东西接触——这才是意外连接的来源。

### 碰撞刺激池

主 agent 从以下 7 类中**随机**选 3 个（每类各选 1 个，分配给 3 个碰撞 agent）：

| 类别 | 示例 | 为什么有效 |
|------|------|-----------|
| 物理现象 | 毛细作用/共振/相变/流体力学 | 物理定律是自然界最深层的结构 |
| 生物机制 | 共生/变态/趋光性/群体智能 | 生物经过亿万年优化，结构极度精炼 |
| 历史事件 | 丝绸之路/黑死病/文艺复兴/冷战 | 历史展示了极端条件下的系统行为 |
| 日常体验 | 等公交/做饭/搬家/做梦 | 日常体验是直觉和隐喻的源泉 |
| 艺术手法 | 蒙太奇/留白/复调/即兴 | 艺术的本质是创造意外连接 |
| 社会制度 | 拍卖/陪审团/货币/护照 | 制度是解决群体问题的结构方案 |
| 游戏机制 | 回合制/随机事件/资源稀缺/隐藏信息 | 游戏是复杂系统的简化模型 |

### 问题结构提取（碰撞前，主 agent 执行）

把核心矛盾拆成可比较的结构。不分析解决方案，只看问题本身的运作方式：

1. 系统边界：矛盾里的"系统"是什么？输入什么？产出什么？
2. 瓶颈：卡点在哪？为什么不能自然解决？
3. 动力学：有什么反馈循环？加剧因素？衰减因素？
4. 脆弱点：系统在什么条件下会崩溃或短路？

输出（喂给碰撞 agent）：
```
系统模型：{输入}→{处理}→{输出}，瓶颈在{环节}
关键动力学：{正/负反馈描述}
脆弱点：{系统在什么条件下失效}
```

### 碰撞 agent prompt

并行 spawn 3 个 sonnet agent（互不可见）：

```
你拿到两样东西：一个核心问题，和一个看起来完全无关的事物。
你的任务是找到它们之间的结构性相似——不是表面类比，是底层运作方式的相似。

核心矛盾：{核心矛盾}
问题结构：系统模型={系统模型}，动力学={关键动力学}，脆弱点={脆弱点}
约束条件：{约束}

碰撞刺激：{刺激物名称}
刺激物的运作机制（简要）：{1-2句描述该刺激物怎么运作}

你的思考步骤：
1. 先想清楚这个刺激物是怎么运作的。它的输入是什么？经过什么处理？输出什么？
   关键机制是什么？什么条件下它会失败？
2. 对比两个系统的结构：
   - 问题系统：{系统模型}
   - 刺激物系统：你刚分析出来的
   找同构点：哪个环节的运作逻辑是一样的？（不是"看起来像"，是"输入→处理→输出链条一样"）
   如果找不到完全同构，找部分同构：某个子环节、某种动力学模式、某种失败条件。
3. 如果找到了结构相似：这个相似指向什么具体的行动/方案/视角？
4. 如果找不到：说出来。"这个刺激物和问题之间没有深层结构连接"也是有效输出。

输出格式：
- 刺激物机制：{输入→处理→输出，一句话}
- 结构相似：{找到了什么 / 没找到}
  同构点：{问题系统的X环节 ≈ 刺激物系统的Y环节，因为...}
- 如果找到：具体指向：{这个相似暗示了什么具体的行动/方案/视角}
- 意外性自检：这个发现，一个充分了解背景的理性分析者会自然想到吗？
  不会 → [意外]  /  会 → [预期内]  /  不确定 → [待验证]
- 不确定处：{哪些判断你不太确定}

禁止：表面类比（"都像流水"）、排列组合（"把A和B结合"）、抽象术语包装
```

### 碰撞 agent 异常

- 超时/失败 → 不重跑，用剩余结果继续
- 3 个都失败 → 换 3 个新刺激，再来一轮（计入回退计数）
- 产出违反用户约束 → 标记"约束违规"，不进入 CAPTURING

---

## CAPTURING：捕捉有价值连接

主 agent 独占。看所有碰撞结果，决定哪些值得追。

### 判断规则

对每个碰撞 agent 的产出：

| 情况 | 判定 | 行动 |
|------|------|------|
| 找到结构相似 + [意外] | 金矿 | 保留，进入 FORGING |
| 找到结构相似 + [预期内] | 有价值但不意外 | 标记"已知方向"，作为背景参考 |
| 找到结构相似 + [待验证] | 不确定 | 保留，进入 FORGING 验证 |
| 没找到结构相似 | 诚实产出 | 记录"此刺激无效"，不勉强 |

### 意外性汇总

- 有 ≥1 个 [意外] 发现 → 进入 FORGING
- 全部 [预期内] 或 "没找到" → **回退到 INCUBATING**（换 3 个新刺激，最多回退 3 轮）
- 回退 3 轮仍无意外发现 → 输出质量不足报告：
  ```
  本次创意捕捉未产出突破性发现。经过 {N} 轮碰撞孵化，所有产出均在预期范围内。
  各碰撞方向及为何预期内：{列出}
  建议换问题框架重新启动 / 引入领域外的人参与 / 先放下过段时间再试。
  ```

### 多方关系（3+实体时触发）

枚举所有双边关系对 → 每对追问：第一性矛盾、不死不休的底线、独立性（去掉第三方仍成立？）→ 检查覆盖率。

---

## FORGING：锻造方案

**此时才引入已有材料和方法论。** 碰撞产出的连接是独立的"种子"，现在用材料验证它、用方法论打磨它。

### 上下文采集（延迟到此刻）

1. 读取用户提到的文件/项目（如果有）
2. 搜索相关领域知识/竞品案例（sonnet agent，如果需要）
3. 搜索异常：工具不可用 → 降级为主 agent 内部知识，标注"[未经外部验证]"

### 碰撞验证

对每个 CAPTURING 保留的发现，用已有材料碰撞验证：

| 碰撞结果 | 含义 | 行动 |
|---------|------|------|
| 别人做过类似的 | 方向对但需差异化 | 追问：我们版本的本质区别是什么？ |
| 和已知事实矛盾 | 要么想法错，要么"已知"要质疑 | 判断哪个更可信。想法更有解释力 → 标记"挑战性假设" |
| 没人做过 | 可能是创新也可能是没人需要 | 深挖可行性 |

### 方法论打磨

此时才选方法论（SCAMPER/TRIZ/六帽/概念融合等，详见 [references/methods.md](references/methods.md)）。
方法论的作用不是"从哪里出发"，而是"怎么把粗糙的想法打磨成可执行的方案"。

### 深度挖掘（可选）

对碰撞存活 + 材料验证后的方向，spawn 1-2 个 sonnet agent 深挖具体化：
```
核心方向：{碰撞发现+碰撞验证结论}
外部材料：{已有内容/搜索结果摘要}

你的任务：
1. 这个方向落地后具体长什么样？（用画面/场景/步骤描述，不用抽象标签）
2. 最可能的失败模式是什么？怎么预防？
3. 第一步是什么？谁来做？

输出：具体化描述 + 失败预防 + 第一步行动
约束：{用户约束}
```

## SHIPPING：交付 + 持久化

### 收敛

1. **常识检验**：每条成果用大白话重说。去掉术语后空了 → 删。必须能回答"具体机制是什么"
2. 去重合并相似创意
3. **事实回检**：用确认清单验证关键逻辑，标记矛盾创意
4. **未验证内容标注**：搜索降级时头部醒目提示

### 持久化

```
~/.claude/skills/brainstorm/data/{YYYYMMDD}-{slug}/
├── 成果.md          # 完整创意清单+历程
└── raw/
    ├── incubating.md  # 碰撞孵化摘要（3个agent的碰撞刺激+产出）
    └── forging.md     # 锻造摘要（材料验证+方法论打磨）
```

slug 冲突追加 `-2`。memory_store 失败不阻塞。

```
memory_store(
  content: "头脑风暴[{主题}]: {核心矛盾}。Top: 1){最佳1} 2){最佳2} 3){最佳3}。意外发现:{最有趣}",
  metadata: { tags: "brainstorm,creative", type: "brainstorm-result" }
)
```

### 输出格式

```
# 创意捕捉成果：{主题}

## 核心矛盾
{一句话}

## 意外发现
1. {碰撞发现}（来自：{刺激物}→结构性相似）
   大白话：{去掉术语后}
   碰撞验证：{别人做过/矛盾/没人做过}

## 有待探索的方向
- {碰撞验证后的方向}

## 碰撞历程
- 碰撞1 ({刺激物}): {找到了/没找到} → {摘要}
- 碰撞2 ({刺激物}): {找到了/没找到} → {摘要}
- 碰撞3 ({刺激物}): {找到了/没找到} → {摘要}
- 最有价值的意外连接：{什么}

## 下一步
- {第一步行动}
```

## 思维原则（所有 agent 必须遵守）

1. **具体化** — 用画面/场景/动作，不用抽象标签。"身份债务""定义权争夺""系统性剥削" = 你在偷懒
2. **自己信吗？** — 换成大白话还站得住才是真话。翻译不出来 = 自己没想清楚
3. **一条真 > 十条不同方向** — 每个碰撞 agent 最多 1 个结构相似发现
4. **没找到就说没找到** — 不勉强连接。"这个刺激物和问题之间没有深层结构连接"也是有效输出

禁止：表面类比 / 排列组合（"把A和B结合"） / 数量配额 / 刻意反转 / 在 INCUBATING 阶段引用已有内容或搜索结果
