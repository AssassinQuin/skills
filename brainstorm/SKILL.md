# 头脑风暴引擎

## 流程总览

```
Phase 0: 需求深挖 → 反驳/追问/逼出真问题
Phase 1: 上下文采集 → 读取已有内容 → 搜索调研(sonnet agent)
🔒 计划确认（需求+事实+搜索+关系矩阵）
Phase 2: 深度发散 → Round 0(找"太顺了") → 视角锁定agents → 交叉验证
Phase 3: 综合收敛 → 常识检验+事实回检 → 保存 → 🔒确认输出
```

每个 Phase 内嵌异常处理，不单独设"边界条件"章节。

---

## Phase 0: 需求深挖（不可跳过）

**核心原则：用户说的大概率不是真需求。** 不追问就开脑洞 = 浪费所有人时间。

### 步骤

1. **先质疑**："你说的X到底想解决什么？""为什么需要？""试过什么？卡在哪？"
2. **逼出 3 个关键信息**：真问题（深层动机）、已有方案/限制、成功标准
3. **犀利但有用**：不是抬杠，每条追问带着一个假设或方向
4. 信息足够清晰 → 直接确认，不多问

### 异常路径（内嵌）

| 情况 | 处理 |
|------|------|
| 用户说"别问了"/"就这样" | 尊重。标注"信息不足，创意基于假设：{列出你的假设}"，在计划确认中展示让用户纠正 |
| 用户回答自相矛盾 | 指出矛盾："你说A又说B，这两个冲突，哪个更准确？"不替用户选 |
| 追问2轮仍模糊 | 停止追问。用已有信息+你的假设填充，在计划确认中标注哪些是假设 |
| 信息足够清晰 | 直接跳到计划确认，不多问 |

### 🔒 计划确认

```
【计划确认】
我理解的问题：{真问题}
约束条件：{已有方案/限制}
成功标准：{什么叫好了}

关键事实：
1. {事实1}
2. {事实2}
...

{标注：以下为假设（用户未确认）：{假设1}、{假设2}}

{搜索方向：{方向1}、{方向2}，用 sonnet agent}
{关系矩阵：{A-B、A-C、B-C...}}
确认 / 补充 / 纠正？
```

用户确认后进入 Phase 2，不再有中间确认。

### 用户纠正机制

- 方向/偏好纠正 → 调整当前方案，继续
- 事实/关系纠正 → 停止。回到关键事实清单重新确认，标记已有创意为"前提有误"，**禁止最小修改——必须从确认过的事实重新推导**
- Phase 2 进行中用户反悔 → 中断当前轮次，回 Phase 0 重新确认需求，已有产出标记"基于旧前提"

### 小说领域增强

检测到小说/角色/世界观信号时，加载 [references/novel-brainstorm.md](references/novel-brainstorm.md)。追问时额外问：设定文件路径？涉及角色及关系？时间线？想给读者什么感受？什么是绝对不能改的？

## Phase 1: 上下文采集

### 读取已有内容

读取用户提到的文件/项目，提取关键信息作为头脑风暴种子。

### 搜索调研

**是否需要搜索**：涉及特定领域知识 → 需要。涉及竞品/参考案例 → 需要。用户有足够上下文 → 不需要。用户说"不用搜" → 跳过。

**执行与异常**：

| 步骤 | 正常 | 异常 |
|------|------|------|
| 广搜（sonnet agent） | 确认方向有用 → 深读 | 空结果 → 换关键词重试一次，仍空 → 标记"该方向无可用资料" |
| 深读 | 提取关键信息 | 结果全不相关 → 换搜索方向或标记"外部资料不可用" |
| 搜索工具不可用 | — | 跳过搜索，在计划确认中标注"未搜索外部资料"，依赖已有信息 |

搜索方向纳入计划确认，不单独确认。

## Phase 2: 深度发散（视角子agent架构）

### 架构

```
主 agent（编排器）
  ├── 选方法论 → 提取关键视角 → 判断哪些"容易偷懒"
  ├── Round 0: spawn 通用探索 agent（找"太顺了"）
  ├── Round 1+: spawn 视角锁定 agent × N（并行，互不可见）
  └── 交叉验证：主 agent 独占（矛盾碰撞+互补发现+盲区扫描）
```

**拆分判断**：
- 拆：天然令人不适、容易敷衍跳过的视角（黑帽批判、假设推翻、利益受损方）
- 不拆：方法论的交叉碰撞价值高于深度（SCAMPER+TRIZ 同时持有才能发现替换恰好解决矛盾）

### 准备：视角提取与拆分决策（30秒）

```
问题类型？→ 选方法论 → 提取视角 → 哪些拆？
├── 改进现有 → SCAMPER → Eliminate/Reverse拆（天然不舒服）
├── 矛盾/两难 → TRIZ → 两边各拆一个agent
├── 全面评估 → 六帽 → 黑帽单独拆
├── 全新方向 → 概念融合 → 两个源领域各拆一个
├── 推翻假设 → 第一性原理 → "推翻最底层"拆
└── ⚠️ 多方关系 → 博弈局面法 → 至少拆出被忽视方
```

**⚠️ 多方关系检测**（3+实体或"三方/多方/博弈/势力"时触发）：
枚举所有双边关系对 → 每对追问：①第一性矛盾 ②不死不休 ③独立性（去掉第三方仍成立？）→ 检查覆盖率，跳过 → 强制补充

**自检**：
1. 不拆会被偷懒跳过吗？不会 → 留主agent
2. 视角间需要互看结果吗？需要 → 不拆
3. 拆出的视角 ≤ 3

### 思维原则（所有 agent 必须遵守）

1. **泡进去** — 感受材料，直到某处让你觉得"不太对"
2. **跟着走** — 追那个不舒服的点，追到底
3. **具体化** — 用画面/场景/动作，不用抽象标签
4. **自己信吗？** — 换成大白话还站得住才是真话
5. **一条真 > 十条不同方向**

禁止：列假设清单逐个推翻 / 数量配额（"至少5个"） / 抽象术语包装空洞（"身份债务""定义权争夺""系统性剥削""存在论层面"=你在偷懒） / 刻意反转

### Round 0：找"太顺了"（强制首轮）

```
Round 0 agent prompt：

你是一个直觉很准的人。读完材料的反应不是"让我分析"，而是"这里有什么不对"。

核心问题：{核心问题}

你的任务：
  1. 过一遍所有材料。哪个部分太顺了？哪个默认接受但其实经不起追问？
  2. 抓住那个"太顺了"的点。追进去——为什么它这么顺？支撑它成立的东西本身站得住吗？
  3. 用最直白的话说出你追到底之后发现的东西。

输出：
  - 哪里"太顺了"（一两句话）
  - 追进去看到了什么（想多长写多长，想明白一件事就够了）
  - 🔥 如果发现让问题变得不同了，用一句话重写问题
```

**Round 0 异常**：
- 找不到"太顺了"的点 → 问题可能太具体或太空泛。告知用户，建议收窄或放宽
- 🔥 触发 → 检查新问题是否与 Phase 0 确认的关键事实一致。不一致 → 需回 Phase 0 补充确认

### Round 1+：视角锁定 agent（并行）

每个 agent 被锁死在自己的框架里——看不到其他视角结论，不会被惯性带跑。

```
视角锁定 agent prompt：

你被锁死在一个特定视角里。只从这个视角深挖到底。
视角：{视角名称+核心问题}
禁令：不允许逃到其他视角。"但从另一个角度看" → 停，回到你的视角。

核心问题：{核心问题}
Round 0 发现：{核心发现}

{视角专属指令：}

[黑帽/批判] 找方案里所有可能出错的地方。追问Round 0发现：如果这个发现本身是错的呢？
  输出：最致命的1-2个问题。

[假设推翻] 找支撑问题成立的最底层假设，质疑它。
  输出：最底层假设 + 如果不成立会怎样。

[利益受损方] 你是利益受损最严重的一方。你的真实处境？你会怎么反抗？
  输出：处境（具体到身体/资源/信息限制）+ 反抗行动。

[领域外嫁接] 从{随机领域}偷结构过来。偷结构不是偷概念。
  输出：偷来的结构 + 怎么嫁接。

[合理化辩护]（小说专用）你是这个方案/势力/反派。不是"扮演坏人"——是你的生存策略从这个角度看完全合理。
  你需要说服中立旁观者：你的选择虽然造成痛苦，但你的处境下没有更好的路。
  输出：你的处境（具体到身体/资源/信息限制）+ 选择逻辑 + 对方哪里没看到你的苦衷。

增量信息：{本轮新事实}

怎么想：泡进去→跟着走→具体化→自己信吗→没走通就说没走通
输出：思考路径（从哪出发→怎么想→到达哪里）+ 🔥真洞察 + ❓不确定处
约束：{用户约束}
```

**并行执行**：多个视角 agent 同时 spawn（互不可见）。

**主 agent 收到结果后**：
1. 展示每个视角的思考路径摘要
2. **常识检验**：用大白话重说，去掉术语空了 → "存疑"
3. agent 说"没走通" → 追问卡点，卡点作为下轮依据
4. 产出弱 → 不重跑，标记"该视角无新发现"+ 原因

**视角 agent 异常**：
- 超时/失败 → 不重跑。关键视角（如唯一批判视角）→ 主 agent 在交叉验证中补做
- 产出违反用户约束 → 标记"约束违规"，不进入交叉验证

### 交叉验证（主 agent 独占）

视角 agent 不可见彼此结果。只有主 agent 做这步。

```
1. 矛盾碰撞：A和B矛盾吗？→ 真矛盾是金矿。区分价值矛盾vs prompt导致的假矛盾：
   假矛盾特征：两边用的前提不同，换前提就一致了。→ 换前提重试
   真矛盾特征：同一前提下仍然对立。→ 追进去

2. 互补发现：A揭示的问题，B恰好能解决？→ 🔗 互补对，最有价值输出

3. 盲区扫描：所有视角都没覆盖的方向？
   关键方向 → 补1个视角agent
   次要方向 → 标记"未探索"

4. Round 0 回检：所有结论加在一起，让"太顺了"变不一样了吗？
   变了 → 🔥 更新问题理解
   没变 → 考虑追加轮次
```

### 轮次管理

```
Round 0（强制）→ 视角提取 → Round 1+（≤3个视角agent）→ 交叉验证
最少2轮（R0+1视角），最多4轮
交叉验证发现关键盲区 → 追加1个视角agent
用户说"继续" → 追加新视角
```

### 自适应解卡

| 状态 | 触发信号 | 注入策略 |
|------|---------|---------|
| 卡壳 | agent产出<100字且无实质内容 | "从{完全无关领域}偷一个结构强制嫁接" |
| 太保守 | 所有创意在搜索引擎前10页能找到 | "删掉它们。假装你是外星观察者，只看到问题本身" |
| 太发散 | 产出>5个方向但无可落地方案 | "找最意外的1个，收敛出3个可落地变体" |
| 框架困住 | 连续2轮产出结构相似 | "回Round 0碎片，选没用过的作为唯一起点" |

### 灵感标记

🔍 反常识 / 🔗 跨领域 / ⚡ 有趣 / ❓ 追问

## Phase 3: 综合收敛 + 持久化

### 收敛步骤

1. **常识检验**：每条成果用大白话重说。去掉术语后空了 → 删。必须能回答"具体机制是什么"
2. **常识检验异常**：全部不通过 → 标注"本次发散质量不足"，展示弱产出+具体不足之处，建议"换问题框架重新启动"
3. 去重合并相似创意
4. **事实回检**：用关键事实清单验证每条关键逻辑，标记矛盾创意
5. **多方关系覆盖率检查**（如触发了关系矩阵）：每对关系必须有"第一性矛盾+不死不休+独立性"三要素
6. 🔒 展示最终成果

### 持久化

**1. 文件保存**：

```
~/.claude/skills/brainstorm/data/{YYYYMMDD}-{slug}/
├── 成果.md          # 完整创意清单+历程
└── raw/
    ├── round-0.md   # 各轮摘要（非完整JSONL）
    └── round-1.md
```

slug 冲突时追加 `-2`、`-3`。raw/ 下只存摘要，不存完整 JSONL。

**2. Memory**：

```
memory_store(
  content: "头脑风暴[{主题}]: {核心问题}。Top: 1){最佳1} 2){最佳2} 3){最佳3}。💡{最有趣灵感}",
  metadata: { tags: "brainstorm,creative", type: "brainstorm-result" }
)
```

memory_store 失败 → 不阻塞。文件已保存，在成果中标注"memory写入失败"。

**3. 项目级记忆**（在项目目录下运行时）：额外加项目 tag。小说场景可同步到灵感库。
