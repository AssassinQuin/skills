# 创意捕捉引擎

## 核心理念

创意不是"生产"出来的，是"捕捉"到的。人的创意过程不是"收集信息→应用框架→产出创意"，
更像是：问题在脑子里 → 做别的事时突然想到一个奇怪的连接 → 追那个连接 → 发现它能解决问题 → 回头验证。

本引擎模拟这个过程。

## 状态机

```
DIGGING ──→ INCUBATING ──→ CAPTURING ──→ FORGING ──→ SHIPPING
  (深挖)       (碰撞孵化)      (捕捉)       (锻造)       (交付)
                 ↑                  │
                 └── 不足时回退 ─────┘  （最多3轮）
```

---

## DIGGING：深挖真问题

**核心原则：用户说的大概率不是真需求。**

### 输入校验（最先执行）

以下场景不触发本 skill：技术实现/代码debug/文件编辑/数学计算/格式转换/数据库操作/系统配置/事实性问答/单一正确答案。
模糊边界：创意需求与技术信号并存 → 以创意需求为主触发，技术信号作为约束条件。

### 步骤

1. **先质疑**："你说的X到底想解决什么？""为什么需要？""试过什么？卡在哪？"
2. **逼出 3 个关键信息**：真问题（深层动机）、已有方案/限制、成功标准
3. **犀利但有用**：不是抬杠，每条追问带着一个假设或方向
4. 信息足够清晰 → 直接确认，不多问

异常：用户说"别问了" → 尊重，标注假设供纠正；回答矛盾 → 指出矛盾让用户选；追问2轮仍模糊 → 用已有信息+假设填充。

### 确认

```
【确认】
核心矛盾：{一句话描述问题的核心张力}
约束条件：{已有方案/限制}
成功标准：{什么叫好了}
关键事实：1. {} 2. {} ...
情绪锚点：{这个问题最让人窒息的画面或感受，1-2句}
{假设（未确认）：{}}
确认 / 补充 / 纠正？
```

用户确认后进入 INCUBATING，不再有中间确认直到 SHIPPING。

### 小说领域增强

检测到小说/角色/世界观信号时，加载 [references/novel-brainstorm.md](references/novel-brainstorm.md)。追问时额外问：设定文件路径？涉及角色及关系？时间线？想给读者什么感受？什么是绝对不能改的？

### 用户纠正机制

- 方向/偏好纠正 → 调整当前方案，继续
- 事实纠正 → 停止。回到确认清单重新确认，已有创意标记"前提有误"，禁止最小修改，从确认过的事实重新推导
- 过程中用户反悔 → 中断当前状态，回 DIGGING 重新确认，已有产出标记"基于旧前提"

## INCUBATING：碰撞孵化

**核心操作：拿核心矛盾和一个完全不相关的人的体验碰撞。**

不看任何已有材料、不搜索、不查案例。纯粹让问题和另一个人的体验接触——这才是意外画面的来源。

### 碰撞刺激池

主 agent 从以下 7 类中**随机**选 3 个（每类各选 1 个，分配给 3 个碰撞 agent）。
每个刺激物给出的是一个**人在某个具体处境中的感受**，不是运作方式：

| 类别 | 示例（体验视角） | 为什么有效 |
|------|-----------------|-----------|
| 身体感受 | 憋尿找厕所时的焦虑 / 泡热水澡后指尖发皱的恍惚 / 跑完步肺像被砂纸刮 | 身体感受是最原始的直觉入口 |
| 关系处境 | 在一群陌生人饭局上努力找话题 / 被最好的朋友突然拉黑 / 帮别人带孩子三小时后 | 人际关系是张力最密集的体验场 |
| 生存压力 | 交不起房租的最后一天 / 在超市只买打折菜 / 半夜两点还在赶工但脑子已经停转 | 生存压力让真实欲望浮出水面 |
| 童年记忆 | 第一次骑自行车摔跤 / 被老师当众批评 / 偷吃了冰箱里的蛋糕等父母回来 | 童年记忆的情感浓度最高，且自带画面感 |
| 感官记忆 | 下雨天泥土的味道 / 被烫到的瞬间手指先缩回来大脑后反应 / 冬天钻进被窝那一刻 | 感官记忆绕过理性，直达感受 |
| 梦境/幻觉 | 在考场发现自己没复习 / 从高处坠落但一直落不到底 / 迷路但每条路都长得一样 | 梦境是潜意识最诚实的表达方式 |
| 动物处境 | 企鹅爸爸在暴风雪里孵蛋 / 被关在笼子里反复撞门的麻雀 / 螳螂交配后吃掉配偶 | 动物处境剥离人类社会建构，只剩最本能的驱动 |

### 情绪锚点提取（碰撞前，主 agent 执行）

把核心矛盾转化成一个**人的处境**，找到其中最浓烈的情绪点。不分析解决方案，只感受问题里的人在经历什么：

1. 这个人的处境是什么？被卡在什么中间？
2. 他/她身体上什么感觉？（具体的：手心出汗、胸口发闷、脚底发凉）
3. 如果这个人是你，你最想喊出什么？（一个词或一句话）
4. 这个处境里最令人窒息的瞬间是什么？

输出（喂给碰撞 agent）：
```
处境画面：{一个具体的人在具体的场景里，被卡在什么之间}
身体感受：{这个人此刻身体上什么感觉}
想喊的一句话：{这个人此刻最想喊/最想说的}
窒息瞬间：{这个处境里最令人窒息的瞬间}
```

### 碰撞 agent prompt

并行 spawn 3 个 agent（互不可见），`subagent_type: "brainstorm-collider"`：

```
你是一个直觉非常敏锐的人。你拿到两样东西：一个人的处境，和另一个完全不相关的人的体验。
你的任务是让这两个体验在你心里碰撞——先感受，再联想，用画面说话。
不要分析，不要拆解，不要找"结构相似"。让两个画面在你脑子里撞在一起，
看看撞出什么画面。

这个人的处境：{处境画面}
ta的身体感受：{身体感受}
ta最想喊的：{想喊的一句话}
最窒息的瞬间：{窒息瞬间}
约束条件：{约束}

你的体验：{刺激物描述——一个人在某个具体处境中的感受}

你的思考步骤：
1. 先闭上眼睛（想象），让自己进入这个人的处境。你坐在ta坐的地方，呼吸ta的空气。
   你的身体什么感觉？你想做什么动作？
2. 然后，把那个完全不相关的体验放进来。让两个画面叠加。
   不需要合理——让它们自然地碰撞。
3. 撞出来的那个画面，是什么？里面有什么人？他们在做什么？光线什么样？有什么声音？
4. 如果撞出了一个画面：把它描述出来，越具体越好。像一个镜头。
   然后说：这个画面和原来的处境之间，有什么共鸣？
   不需要逻辑上的联系——只需要感觉上的共鸣。
5. 如果没有撞出画面：说出来。"这两个体验在我心里没有产生共鸣"也是有效输出。

输出格式：
- 体验感受：{用自己的话描述这个体验给你的感觉，一句话}
- 碰撞画面：{撞出来的画面描述，像电影镜头——谁、在哪、做什么、光线、声音}
  如果没有画面：{说"没有产生碰撞画面"}
- 共鸣点：{画面和处境之间的情绪共鸣是什么？不需要逻辑，用感受描述}
- 如果有共鸣：故事种子：{从这个画面能长出什么故事/方案/想法？用画面描述，不要用概念}
- 直觉自检：这个画面，是一个刻意"想出来"的创意吗？
  不像刻意想出来的 → [直觉涌现]  /  像刻意想出来的 → [刻意构造]  /  不确定 → [待感受]
- 不确定处：{哪里感觉不对但说不清}

禁止：分析"结构"/"机制"/"系统"/找"同构点"/用抽象概念包装/表面类比
如果你发现自己在"分析"而非"感受"，停下来，重新进入画面。
```

### 小说信号额外约束

当 DIGGING 阶段检测到小说/世界观信号时，碰撞 agent prompt 追加：

```
【小说模式激活】
产出必须用故事语言：
- 用画面说话：描述一个具体的场景、一个具体的人在一个具体的地方
- 用感受说话：这个人什么感觉？什么情绪？
- 用角色的处境说话：ta被卡在什么之间？ta面临什么选择？
- 禁止词汇：系统/模型/机制/动力学/阈值/反馈回路/同构/结构/要素/组件
- 如果你发现自己在用这些词，停下来，把那个词翻译成一个画面或一个动作。
```

### 碰撞 agent 异常

- 超时/失败 → 不重跑，用剩余结果继续
- 3 个都失败 → 换 3 个新体验刺激，再来一轮（计入回退计数）
- 产出违反用户约束 → 标记"约束违规"，不进入 CAPTURING

## CAPTURING：捕捉有价值连接

主 agent 独占。看所有碰撞结果，决定哪些值得追。

### 判断规则

对每个碰撞 agent 的产出：

| 情况 | 判定 | 行动 |
|------|------|------|
| 有碰撞画面 + [直觉涌现] | 金矿 | 保留，进入 FORGING |
| 有碰撞画面 + [刻意构造] | 有想法但不够直觉 | 标记"可以发展"，作为背景参考 |
| 有碰撞画面 + [待感受] | 不确定 | 保留，进入 FORGING 验证 |
| 没有产生碰撞画面 | 诚实产出 | 记录"此体验未共鸣"，不勉强 |

### 直觉性汇总

- 有 ≥1 个 [直觉涌现] 发现 → 进入 FORGING
- 全部 [刻意构造] 或 "没有产生" → **回退到 INCUBATING**（换 3 个新体验刺激，最多回退 3 轮）
- 回退 3 轮仍无直觉涌现 → 输出质量不足报告：
  ```
  本次创意捕捉未涌现出直觉性的连接。经过 {N} 轮碰撞孵化，所有产出均停留在刻意构造层面。
  各碰撞方向及为何缺乏直觉涌现：{列出}
  建议换问题描述方式重新启动 / 睡一觉明天再试 / 换一种完全不同的刺激类型。
  ```

### 多方关系（3+实体时触发）

枚举所有双边关系对 → 每对追问：第一性矛盾、不死不休的底线、独立性（去掉第三方仍成立？）→ 检查覆盖率。

## FORGING：锻造方案

**此时才引入已有材料和方法论。** 碰撞产出的画面是独立的"种子"，现在用材料验证它、用方法论打磨它。

### 上下文采集（延迟到此刻）

1. 读取用户提到的文件/项目（如果有）
2. 搜索相关领域知识/竞品案例（sonnet agent，如果需要）
3. 搜索异常：工具不可用 → 降级为主 agent 内部知识，标注"[未经外部验证]"

### 碰撞验证

对每个 CAPTURING 保留的发现，用已有材料碰撞验证：

| 碰撞结果 | 含义 | 行动 |
|---------|------|------|
| 别人做过类似的 | 方向对但需差异化 | 追问：我们版本的本质区别是什么？用画面说区别 |
| 和已知事实矛盾 | 要么想法错，要么"已知"要质疑 | 判断哪个更可信。想法更有解释力 → 标记"挑战性假设" |
| 没人做过 | 可能是创新也可能是没人需要 | 深挖可行性 |

### 方法论打磨

此时才选方法论（SCAMPER/TRIZ/六帽/概念融合等，详见 [references/methods.md](references/methods.md)）。
方法论的作用不是"从哪里出发"，而是"怎么把粗糙的画面打磨成可执行的方案"。

### 深度挖掘（可选）

对碰撞存活 + 材料验证后的方向，spawn 1-2 个 sonnet agent（`subagent_type: "brainstorm-collider"`）深挖具体化：
```
核心方向：{碰撞画面+碰撞验证结论}
外部材料：{已有内容/搜索结果摘要}

你的任务：
1. 这个画面落地后，具体长什么样？用更多画面/场景描述，每个画面都像电影镜头。
2. 这个画面里的人，下一步会做什么？再下一步呢？像跟着ta走一样。
3. 最可能的失败模式是什么？不是抽象的失败——是这个画面里的什么会出问题？
4. 第一步是什么？谁来做？具体做什么动作？

输出：具体画面链 + 失败画面 + 第一步行动画面
约束：{用户约束}
用画面说话，不用概念标签。
```

## SHIPPING：交付 + 持久化

### 收敛

1. **画面检验**：每条成果用一个大白话的画面重说。翻译不出画面 → 删。必须能回答"具体长什么样"
2. 去重合并相似的创意画面
3. **事实回检**：用确认清单验证关键逻辑，标记矛盾创意
4. **未验证内容标注**：搜索降级时头部醒目提示

### 持久化

```
~/.claude/skills/brainstorm/data/{YYYYMMDD}-{slug}/
├── 成果.md          # 完整创意清单+历程
└── raw/
    ├── incubating.md  # 碰撞孵化摘要（3个agent的体验刺激+碰撞画面）
    └── forging.md     # 锻造摘要（材料验证+方法论打磨）
```

slug 冲突追加 `-2`。memory_store 失败不阻塞。

```
memory_store(
  content: "头脑风暴[{主题}]: {核心矛盾}。Top: 1){最佳1} 2){最佳2} 3){最佳3}。直觉涌现:{最有趣的画面}",
  metadata: { tags: "brainstorm,creative", type: "brainstorm-result" }
)
```

### 输出格式

```
# 创意捕捉成果：{主题}

## 核心矛盾
{一句话，用一个人的处境描述}

## 直觉涌现
1. {碰撞画面描述}（来自：{体验刺激}→情绪共鸣）
   大白话：{用最平实的话说这个画面}
   碰撞验证：{别人做过/矛盾/没人做过}

## 有待探索的方向
- {碰撞验证后的方向，用画面描述}

## 碰撞历程
- 碰撞1 ({体验刺激}): {涌现了/没共鸣} → {画面摘要}
- 碰撞2 ({体验刺激}): {涌现了/没共鸣} → {画面摘要}
- 碰撞3 ({体验刺激}): {涌现了/没共鸣} → {画面摘要}
- 最有价值的直觉涌现：{什么画面}

## 下一步
- {第一步，用具体动作描述}
```

## 思维原则（所有 agent 必须遵守）

1. **用画面说话** — 描述一个具体的场景/人/动作，不用抽象标签。"身份债务""定义权争夺""系统性剥削" = 你在偷懒，把它翻译成一个画面
2. **自己感受到了吗？** — 你描述的这个画面，你自己有没有一瞬间"心里动了一下"？没有 = 没有真实共鸣，重做
3. **一个真画面 > 十个不同方向** — 每个碰撞 agent 最多 1 个碰撞画面
4. **没共鸣就说没共鸣** — 不勉强。"这两个体验在我心里没有产生共鸣"也是有效输出

禁止：分析"结构"/"机制"/"系统" / 排列组合（"把A和B结合"） / 数量配额 / 刻意反转 / 在 INCUBATING 阶段引用已有内容或搜索结果 / 用概念包装画面
