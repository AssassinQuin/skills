## 1. SCAMPER

**最适合**：改进现有方案/设定/产品
**学习曲线**：低

对目标逐项过7个动词，每个动词回答一个核心问题：

| 动词 | 问题 | 引导提示 |
|------|------|---------|
| **Substitute** | 什么能替换？ | 材料/角色/流程/视角，换一个会怎样？ |
| **Combine** | 什么能合并？ | 两个不相关的功能/角色/概念组合？ |
| **Adapt** | 什么能借鉴？ | 其他领域/文化/时代的类似问题怎么解决？ |
| **Modify** | 什么能放大/缩小？ | 夸张10倍会怎样？缩小到极简呢？ |
| **Put to another use** | 还能用于什么？ | 目标对象换个场景/角色/目的？ |
| **Eliminate** | 什么能删除？ | 去掉最核心的元素会怎样？去掉最不起眼的呢？ |
| **Reverse** | 什么能反转？ | 顺序反转？角色互换？因果倒置？ |

**操作**：列出7行，每行至少产出2个方向。Eliminate 往往产出最大创新。

## 2. TRIZ

**最适合**：遇到矛盾/两难/此消彼长
**学习曲线**：陡峭（但精简版可直接使用）

### 核心机制

识别**矛盾**，然后用发明原理化解：

**常见矛盾模式**（精简版）：
- 想要A但又想要-A（如：要自由又要安全）
- 改善X导致Y恶化（如：加速导致质量下降）
- 系统有相互冲突的需求

**10个最实用的发明原理**：
1. **分割** — 将整体拆成独立部分
2. **抽取** — 取出关键部分单独使用
3. **局部质量** — 不同部分不同对待
4. **嵌套** — 一个结构包含另一个
5. **反向** — 做与常规相反的事
6. **预先作用** — 提前准备好
7. **动态化** — 让静态变动态
8. **周期性作用** — 连续变脉冲
9. **变色龙** — 改变颜色/透明度/属性
10. **自服务** — 让对象自己完成功能

**操作**：
1. 写出矛盾："想要__但又__"
2. 从10个原理中找3个最相关的
3. 每个原理生成1-2个具体方案

## 3. 六顶思考帽

**最适合**：全面评估/避免盲区/团队决策
**学习曲线**：低

6个角色依次审视同一个创意/方案：

| 帽子 | 角色 | 核心问题 | 典型输出 |
|------|------|---------|---------|
| ⬜ 白帽 | 事实分析师 | 我们确切知道什么？缺少什么数据？ | 事实清单/信息缺口 |
| 🔴 红帽 | 直觉感应器 | 我的感觉是什么？直觉告诉什么？ | 情绪反应/直觉判断 |
| ⬛ 黑帽 | 风险审计师 | 什么地方可能出错？最坏情况？ | 风险清单/失败模式 |
| 🟡 黄帽 | 乐观探矿者 | 最大的价值是什么？为什么能行？ | 机会清单/价值点 |
| 🟢 绿帽 | 创意发电机 | 还有什么替代方案？能否完全不同？ | 新方案/替代路径 |
| 🔵 蓝帽 | 流程指挥官 | 我们学到了什么？下一步？ | 结论/行动计划 |

**操作**：对每个关键创意，依次戴6顶帽子各产出1-2条。黑帽+绿帽组合最有价值（找风险→找替代）。

## 4. 侧向思维

**最适合**：突破思维定势/范式转换
**学习曲线**：中

### 技术清单

**PO挑衅法**：
1. 写出一个荒谬/矛盾的陈述（PO: ___）
2. 从中提取有用的"跳动点"
3. 将跳动点转化为实际方案

**随机输入法**：
1. 从任意来源获取一个随机词（字典/新闻/物体名）
2. 列出这个词的5个属性
3. 将每个属性强制关联到当前问题
4. 找到最意外的关联并发展

**挑战假设法**：
1. 列出关于问题的所有"理所当然"假设
2. 逐个质疑："这真的是必须的吗？"
3. 取消假设后重新构思

**概念扇**：
1. 问题 → 这个问题属于什么更大类的问题？
2. 大类问题 → 还有什么其他子问题？
3. 子问题 → 各自的解决方案是什么？

## 5. 形态分析

**最适合**：多维度/系统性探索所有可能
**学习曲线**：中

### 操作步骤

1. 将问题拆解为3-6个**关键维度**
2. 每个维度列出3-5个**可能值**
3. 构建矩阵：
   ```
         维度1    维度2    维度3
         ─────────────────────────
   值A   │ 选项1A   选项2A   选项3A
   值B   │ 选项1B   选项2B   选项3B
   值C   │ 选项1C   选项2C   选项3C
   ```
4. 从每列选一个值组合 → 总组合数 = 各维度值数的乘积
5. 筛选最有趣的5-10个组合深入发展

**关键**：组合的意外性是创意的来源。故意选择"看起来不搭"的组合。

## 6. 第一性原理

**最适合**：推翻既有假设/根本性创新
**学习曲线**：高

### 四步流程

1. **确定真正问题** — 用户说的是表面问题，真正问题是什么？（"更快的马"→"从A到B的交通"）
2. **分解为基本元素** — 把问题拆到不能再拆的最小单元
3. **区分假设与事实** — 哪些是"大家都这么认为"的假设？哪些是不可动摇的事实？
4. **从零重构** — 只基于事实，用全新的方式重新组装

**三大障碍提醒**：
- 认知负荷：大脑走捷径是本能，刻意走慢路
- 确认偏见：主动寻找反面证据
- 对未知的恐惧：推翻假设会不舒服，这是正常的

## 7. 逆向思维

**最适合**：找到盲区/发现被忽视的方向
**学习曲线**：低

### 三种模式

**结果反推**：
1. 描述理想的最终状态
2. 从终点往回推：要达到这个状态，前一步必须是什么？
3. 一直推到当前状态

**反向问题**：
1. 不问"怎么解决问题"，问"怎么让问题更糟"
2. 把"更糟方案"全部反转
3. 反转后的方案往往比正向思考更有创意

**角色互换**：
1. 如果我是对手/用户/旁观者/十年后的自己，我会怎么看？
2. 从每个视角重新定义问题

## 8. 仿生学

**最适合**：从自然界寻找跨物种灵感
**学习曲线**：中

### 5步流程

1. **定义功能需求**：需要什么能力？（如：在极端环境下生存）
2. **翻译为生物学语言**：自然界什么生物面临类似挑战？
3. **发现自然模型**：找到3-5种生物的策略
4. **抽象核心策略**：提取可迁移的原理（非照搬外形）
5. **应用到当前问题**：将原理适配到目标场景

## 9. 概念融合

**最适合**：跨领域嫁接产生新物种
**学习曲线**：低-中

### Fauconnier-Turner 概念融合法

1. 选择两个**不同领域**的概念A和B
2. 列出A的核心属性和B的核心属性
3. 找到A和B的**共享结构**（抽象层面的共同点）
4. 在共享结构上创造**融合体**：既不是A也不是B，而是新的C
5. 发展C的独特属性（A和B都没有的新特征）

## 10. 角色风暴

**最适合**：换位思考/打破自我中心
**学习曲线**：低

### 操作

选择3-5个**完全不同的角色**，从每个角色的视角回答同一个问题：

角色选择建议：
- 极端用户（最挑剔/最需要的人）
- 外行（完全不懂的人）
- 专家（领域权威）
- 反对者（最不想要这个的人）
- 未来人（10年后的视角）
- 其他行业从业者（医生看教育/建筑师看软件）

每个角色至少产出1条独特视角。
