# Doc Co-Authoring

Guide users through a structured workflow for co-authoring documentation, proposals, technical specs, and decision records. The AI acts as a co-author, not a ghostwriter — the user drives content, the AI drives structure.

## Trigger

写文档、co-author、proposal、技术方案、设计文档、RFC、decision record

## Quick Start

```
/doc-coauthoring 帮我写一个微服务拆分的技术方案
/doc-coauthoring 我需要写一个 API 设计提案
/doc-coauthoring 合作写一个季度 OKR 文档
```

## Workflow

1. **Stage 1: Context Gathering** — Initial questions → info dumping → clarify scope
2. **Stage 2: Refinement & Structure**
   - Clarifying questions
   - Brainstorming structure options
   - Curation (user selects)
   - Gap check
   - Drafting
3. **Stage 3: Review & Polish** — Iterate on draft with user feedback

## Key Principles

- User owns the content; AI owns the structure
- Never ghostwrite — every section needs user input
- Progressive disclosure: don't overwhelm with the full template upfront
- Gap check ensures nothing important is missed
