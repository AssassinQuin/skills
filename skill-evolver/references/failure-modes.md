# 7 Skill Failure Modes (Diagnostic Reference)

> 来源: [skill-craft](https://github.com/3stoneBrother/skill-craft) 方法论提炼。
> 用于 baseline 评估和 audit 诊断分类。

## 模式定义

### FM1: 约束衰减
对话变长后，SKILL.md 中的规则被模型逐渐忽略。
- **检测**: 连续使用 5+ 轮后行为是否偏离规则
- **缓解**: 硬约束写在 frontmatter；关键规则加自检标记（"每次输出前自检此条"）

### FM2: 工具选择漂移
模型在工具超时/失败后，悄悄换成更熟悉的替代工具，不再回来。
- **检测**: 检查 SKILL.md 是否有显式工具优先级表 + 降级条件
- **缓解**: 写明首选/降级工具对，要求标注降级原因

### FM3: 输出膨胀
模型输出远超需求，挤占上下文窗口，加速约束衰减。
- **检测**: 检查是否有明确的输出长度限制和禁止输出列表
- **缓解**: "输出不超过 N 行" + 列出禁止输出的模式（开场白、复述等）

### FM4: 依赖链断裂
多步流程中，步骤间的数据量不匹配（Step 1 找到 29 个，Step 3 只处理 20 个）。
- **检测**: 检查流程步骤是否有 assert 式计数（"已完成数 == 应完成数"）
- **缓解**: 每步输出计数，下一步验证

### FM5: 并行孤岛
多个子 agent 并行产出互相矛盾的结论，主 agent 不做校验直接合并。
- **检测**: 检查子 agent 合并规则是否有去重 + 一致性校验
- **缓解**: 合并步骤显式要求：去重 → 一致性检查 → 计数验证

### FM6: 触发模糊
Skill 在不该触发时触发（用户只是提问，Skill 启动了完整流程）。
- **检测**: 检查是否有 "DO NOT trigger" 段落和歧义处理规则
- **缓解**: 显式列出触发条件 + 不触发条件 + 歧义时的默认行为

### FM7: 幻觉填充
工具未返回结果时，模型基于记忆/经验编造答案而非报"未找到"。
- **检测**: 检查是否有 "无来源 = 不输出" 规则
- **缓解**: 要求标注来源（文件路径+行号），未查到必须输出 "UNABLE TO ASSESS"

## 在进化流程中的使用

### baseline 评估
用 FM1-FM7 作为诊断分类框架，扫描 SKILL.md 结构缺口：
- 缺工具优先级表 → FM2 风险
- 缺输出约束 → FM3 风险
- 缺 DO NOT trigger → FM6 风险
- 缺来源标注要求 → FM7 风险

### audit 诊断
audit 报告中按 FM 分类发现的问题，便于追踪同类问题的修复进展。

### 策略匹配
exploration 阶段选择策略时，优先匹配 FM 风险最高的维度：
- FM1/FM3 高风险 → 考虑 S1(结构强化) 或 S2(约束加固)
- FM6 高风险 → 考虑 S3(边界收窄)
- FM4/FM5 高风险 → 考虑 S4(流程加固)
