# Skill 评估 Rubric

评估维度共 8 项，每项 1-5 分。总分 40 分，用于 P4 深读阶段对候选 Skill 打分。

## 评分标准

### D1: 问题定义（Problem Clarity）

| 分数 | 标准 |
|------|------|
| 5 | 一句话说清解决什么问题、谁会用、为什么值得装 |
| 3 | 能看出解决什么，但目标用户或场景模糊 |
| 1 | 读完后不知道这 Skill 到底干嘛的 |

### D2: 触发精准度（Trigger Precision）

| 分数 | 标准 |
|------|------|
| 5 | frontmatter description 覆盖所有触发场景，无误触发 |
| 3 | 基本能触发，但部分场景缺失或有歧义 |
| 1 | 触发词模糊，容易误触或漏触 |

### D3: 工作流可执行性（Workflow Executability）

| 分数 | 标准 |
|------|------|
| 5 | 步骤具体、有条件分支、可直接执行，无需猜测 |
| 3 | 有步骤但不完整，某些环节需要 Agent 自行发挥 |
| 1 | 只有抽象指导（"根据情况优化"），无具体操作 |

### D4: 失败模式覆盖（Failure Mode Coverage）

| 分数 | 标准 |
|------|------|
| 5 | 显式列举已知失败路径 + 处理方式 + 停手条件 |
| 3 | 有部分错误处理，但不系统 |
| 1 | 无失败模式，假设一切顺利 |

### D5: 依赖与安全（Dependencies & Safety）

| 分数 | 标准 |
|------|------|
| 5 | 依赖最少化；权限声明清晰；无安全风险操作 |
| 3 | 依赖合理但有隐含要求；权限边界基本清晰 |
| 1 | 依赖重或需要敏感权限；未声明安全边界 |

### D6: 可复现性（Reproducibility）

| 分数 | 标准 |
|------|------|
| 5 | 有示例输入输出 + 测试 prompt + 可 3 分钟跑通 |
| 3 | 有示例但不完整，需要额外摸索 |
| 1 | 无示例，或示例无法直接运行 |

### D7: 文档质量（Documentation Quality）

| 分数 | 标准 |
|------|------|
| 5 | README 10 秒内说明价值；有 badge/GIF/截图；安装一步到位 |
| 3 | 文档有但冗长或缺少关键信息（安装/配置） |
| 1 | 无 README 或 README 无实质内容 |

### D8: 架构健康（Architecture Health）

| 分数 | 标准 |
|------|------|
| 5 | SKILL.md < 500 行；references/ 渐进披露；文件结构清晰 |
| 3 | SKILL.md 偏大但可接受；结构基本合理 |
| 1 | SKILL.md 臃肿（>1000 行）；单文件塞入所有内容 |

## 评级映射

| 总分 | 评级 | 含义 |
|------|------|------|
| 34-40 | A | 生产级，可直接采纳 |
| 27-33 | B | 良好，小调整后可用 |
| 20-26 | C | 可用但有明显短板 |
| 13-19 | D | 粗糙，需要大量改进 |
| < 13 | F | 不建议使用 |
