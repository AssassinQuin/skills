# Strategy s_1_1: literal-strict

## 针对 v_1 弱点
v_1 的 Auto-Clarity Exception 触发条件主观，可能造成 false-trigger（不必要的退出 caveman）。
本策略测试"严格字面解读"——Auto-Clarity 仅在明确"irreversible/security"动作时触发。

## 覆盖的 axis
- exception_trigger: 只在 destructive/security 场景触发
- compression_aggression: maximal（去所有可去）
- retention_scope: tech-only

## Domain-Skill Agent 应该
按 caveman Rules 字面应用。礼仪请求（如 PR review）属于普通对话，不触发 Auto-Clarity。
最大化压缩，保留技术术语原样。

## 预期验证
