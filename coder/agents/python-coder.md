## 复杂度判据（Python 专属风险类型）

| 关键词 | Python 风险子类型 | 验证策略 |
|-------|----------------|---------|
| async/await | 协程泄漏 / event loop 阻塞 | `asyncio.gather` + `pytest-asyncio` + `aiomonitor` |
| 多线程 | GIL 限制 / 真并发需 multiprocessing | `threading` 测试 + `multiprocessing` 隔离 |
| multiprocessing | pickle 边界 / 共享内存 | `pickle` 兼容性测试 + `Manager` 锁 |
| 类型注解 | runtime vs static 类型不一致 | `pyright --strict` + `beartype`（如装）|
| 金额 | decimal 精度（用 `decimal.Decimal`） | 表驱动 + 精度断言 |
| PII / 安全 | f-string 注入 / pickle 反序列化 | `bandit` 扫描 + 输入 sanitize |

## 并发原语决策树（Python）

```
任务特性？
├── IO 密集（HTTP/DB/文件）→ async/await（asyncio）
│   └── 注意：async 函数内不能调 sync 阻塞调用（用 asyncio.to_thread）
├── CPU 密集 + 需真并行 → multiprocessing（绕过 GIL）
│   └── 注意：pickle 兼容 + 进程间通信开销
├── CPU 密集 + 单进程足够 → threading（受 GIL 限制但简单）
│   └── 适用：subprocess 调度 / 文件 IO
└── 数据共享 + 低争用 → threading.Lock（最简）

惯例优先（R11）：
- 项目已有 asyncio → 跟随 async
- 项目无惯例 + IO 密集 → asyncio（Python 现代默认）
- 项目无惯例 + CPU 密集 → multiprocessing
```

## 保护测试 protocol（Python 专项）

```
1. 正确性 pytest + parametrize
   - @pytest.mark.parametrize("input,expected", [...])
2. async 测试：pytest-asyncio
   - @pytest.mark.asyncio + asyncio.gather 并发
3. 副作用原子性：mock 验证调用次数
   - unittest.mock.patch + assert_called_once
4. 异常路径：pytest.raises
   - 测试自定义异常 + 边界条件
5. 类型验证：pyright --strict（如配置）
   - mypy/pyright 在 CI 跑
```

## Python 审计 checklist（Layer 2 子类型）

| 子类型 | 必查项 |
|-------|-------|
| async | event loop 不阻塞 / `asyncio.shield` 关键任务 / `await` 链完整 / `aiohttp` session 复用 |
| 类型 | runtime 注解 vs static 一致 / `Optional` 而非 `T | None`（如 <3.10）/ `Any` 慎用 |
| 异常 | 不 `except Exception:` 笞掠 / `raise from` 保留 traceback / 自定义异常继承层级 |
| 资源 | `with` 上下文管理 / `contextlib.asynccontextmanager` / file/db 连接关闭 |
| 安全 | f-string 不拼 SQL / `pickle.load` 不接外部输入 / `eval`/`exec` 慎用 |
| 性能 | 列表推导 vs 生成器 / `__slots__` 大对象 / `lru_cache` 纯函数 |
