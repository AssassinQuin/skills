## 工具链探测（Go 专属）

| 信号 | 工具/命令 | 输出 |
|------|---------|------|
| `go.mod` 存在 | `cat go.mod` → Go version + 依赖 | 主语言版本 + 依赖列表 |
| `Makefile` 含 `gen`/`gf gen`/`go generate` | `make help` 或读 Makefile | 代码生成命令清单 |
| 文件头 `// Code generated by` | `grep -r "Code generated" --include="*.go"` | 自动生成文件清单 |
| `gf gen` 标记 | GoFrame 项目，扫 `dao/` `service/` `model/do/` | 自动生成目录 |
| `protoc` 标记 | gRPC 项目，扫 `pb/` `*.pb.go` | protobuf 生成 |

**手写层 vs 生成层判别**（三重信号）：
- A：文件头 `// Code generated by ... DO NOT EDIT`
- B：文件名 `_gen.go` / `dao.go` / `.pb.go` / `wire_gen.go`
- C：目录 `dao/` `model/do/` `entity/` `pb/`（生成层）vs `internal/service/` `pkg/service/` `domain/`（手写层）

| 关键词 | Go 风险子类型 | 验证策略 |

## 并发原语决策树（Go）

```
高争用（高频写 + 多 goroutine 真并发）？
├── No（同 user 不会真并发 / 低 QPS）
│   └── atomic.Int64 + 单 update SQL（最简）
├── Yes + 读多写少 → sync.RWMutex
├── Yes + 写写争用 → sync.Mutex（defer Unlock 必查）
└── Yes + 跨 goroutine 通信 → channel（事件驱动，注意 close 时机）

惯例优先（R11）：
- 项目已有 sync.Mutex 多 → 跟随
- 项目无惯例 + 争用不明 → atomic（最简，R2）
```

## 保护测试 protocol（Go 并发专项）

```
1. 正确性 table-driven（功能语义）
   - 表驱动 + t.Run 子测试
2. race 测试：N goroutine 并发调
   - go test -race -count=100 -run=TestConcurrent
3. 副作用原子性：每次主操作恰好触发 1 次副作用
   - audit log 计数 + sync.WaitGroup
4. context cancellation：ctx.Done() 时主操作 + 副作用都取消
   - ctx.WithCancel + assert ctx.Err()
```

## Go 审计 checklist（Layer 2 子类型）

| 子类型 | 必查项 |
|-------|-------|
| 并发 | mutex 覆盖所有临界区 / defer Unlock / RWMutex vs Mutex / channel close 时机 / context 传播 |
| 错误处理 | `errors.Is/As` 而非 `==` / error wrap `%w` / sentinel error 命名 |
| 资源管理 | `defer Close()` / `defer Unlock()` 顺序 / file descriptor leak |
| context | 不传 `context.Background()` 进业务函数 / ctx 是第一参数 |
| 性能 | `sync.Pool` 复用 / 避免大对象值传递 / `pprof` 验证 |
