# 验证循环

## 验证链

每阶段改动后必须通过：

```
go_diagnostics (快) → go build ./... (中) → go test ./... (慢)
```

| 阶段 | 工具 | 时机 |
|------|------|------|
| 即时 | `go_diagnostics` | 每次 Edit/Write 后立即验证（零开销） |
| 阶段 | `go build` | 每阶段结束后验证编译 |
| 最终 | `go test` | 最终验证功能正确性 |
| 全量 | `go vet` | 最终全量静态分析 |

## 区分预存测试失败

不要用 `git stash` 测试（同包新旧文件会冲突）。改用：

```bash
# 检查测试文件最近是否被其他 commit 修改但断言未更新
git log --oneline -5 -- path/to/test_file.go
```

如果最近 commit 修改了被测代码但测试断言未同步 → 预存问题，不计入当前改动失败。

## 反模式

| 反模式 | 正确做法 |
|--------|----------|
| 大爆炸式重构 | 分阶段，每阶段独立验证 |
| stash 测试预存失败 | 查 git log 识别预存问题 |
