## 两大短板

### 短板 1: 指令具体性（9.0/15）

**核心问题**：面向人类审阅者写作，非面向 LLM 执行者。多处依赖常识推理而非精确参数。

**改进方向**：
1. 伪代码 → 可填充模板（`{{变量}}` 标记）
2. "相似>0.85" → 具体判定策略
3. 规范粒度 → 补2-3个真实示例
4. "中等改动" → 行数范围/文件数阈值

### 短板 2: 实测表现（15.0/25）

**核心问题**：quick 模式流程散落、歧义多，最常用的简单任务反而执行不可预测。

**改进方向**：
1. quick 模式独立精简流程图
2. Bug Fix quick 同类排查时机明确化
3. 补充 quick 产出约定（无 task-plan、无测试审计）
4. 首次使用标准提示语格式
